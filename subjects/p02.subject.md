# โจทย์ที่ 2: ระบบคำนวณระยะเวลาเหตุการณ์ (Interface & Computed Fields)

**Path:** `GET /incidents`

**ผลลัพธ์:** ข้อมูลเหตุการณ์ทั้งหมดจากไฟล์ `data/incidents.json` พร้อม field พิเศษ `resolutionDays`

**ไฟล์ที่ต้องสร้าง:**

- `src/incident/incident.interface.ts` (interface `IIncident`)
- สร้างไฟล์ `data/incidents.json` ที่ root ของโปรเจกต์ (เพื่อนบ้านกับ package.json) โดยมีข้อมูลทั้งหมดดังนี้:
  ```json
  [
    {
      "id": "1",
      "incidentName": "SQL_INJECTION",
      "status": "RESOLVED",
      "attackerIp": "192.168.1.10",
      "severity": "MEDIUM",
      "detectedAt": "2024-01-01",
      "resolvedAt": "2024-01-10"
    },
    {
      "id": "2",
      "incidentName": "DDOS_ATTACK",
      "status": "ACTIVE",
      "attackerIp": "Unknown Botnet",
      "severity": "HIGH",
      "detectedAt": "2024-02-15",
      "resolvedAt": null
    }
  ]
  ```
  **(หมายเหตุ: ห้ามนักเรียนแก้ไขข้อมูลในไฟล์ `data/incidents.json` ด้วยตัวเอง)**

**ไฟล์ที่ต้องแก้ไข (จากข้อ 1):**

- `src/incident/incident.service.ts` (เพิ่ม method `findAll()`)
- `src/incident/incident.controller.ts` (เพิ่ม endpoint `GET /incidents`)

**เงื่อนไข:**

- Interface `IIncident` ต้องมี Attributes ตามโครงสร้างใน `data/incidents.json`
- Method `findAll()` ต้องอ่าน JSON ไฟล์จาก `data/incidents.json`
- ก่อนที่จะ Retun ค่า ให้ทำการ Map ข้อมูลเพื่อเพิ่ม field พิเศษชื่อ `resolutionDays` เข้าไปในแต่ละ object
- `resolutionDays` (ตัวเลข) ให้คำนวณจำนวนวันจาก `detectedAt` ไปถึง `resolvedAt`
- ถ้า `resolvedAt` เป็น `null` (เหตุการณ์ยังไม่จบ) ให้ตั้ง `resolutionDays` เป็น `-1`

**ตัวอย่างการเรียก API:**

```bash
curl http://localhost:3000/incidents
```

**ตัวอย่างผลลัพธ์:**

```json
[
  {
    "id": "1",
    "incidentName": "SQL_INJECTION",
    "status": "RESOLVED",
    "detectedAt": "2024-01-01",
    "resolvedAt": "2024-01-10",
    "resolutionDays": 9
  },
  {
    "id": "2",
    "incidentName": "BRUTE_FORCE",
    "status": "ACTIVE",
    "detectedAt": "2024-02-15",
    "resolvedAt": null,
    "resolutionDays": -1
  }
]
```

**หมายเหตุ:**

- ถือว่าสูตรหาจำนวนวันคือ `(ปลายทาง - ต้นทาง) / เวลาใน 1 วันตามหน่วย milliseconds` หรือใช้ library `Date` ช่วย
- ข้อนี้ทดสอบการกำหนด Interface, อ่านไฟล์ภายนอก, และแปลง (Transform) ข้อมูลก่อนเสิร์ฟ

**ข้อห้าม:**

1. ห้ามใช้ `process.exit()` มิเช่นนั้นจะตัดคะแนนเป็น 0 ทันที
2. ห้ามแก้ไข file `incidents.json` หากใช้เพื่อทดสอบเสร็จแล้วให้แก้ไขเป็นข้อมูลเดิมหรืออย่า push ขึ้น git repository.
