# โจทย์ที่ 1: สรุปสถานะเหตุการณ์ด้านความปลอดภัย (Data Aggregation & Reduction)

**Path:** `GET /incidents/summary`

**ผลลัพธ์:** JSON object แสดงผลรวมจำนวนเหตุการณ์แยกตามสถานะ

**ไฟล์ที่ต้องสร้าง:**

- `src/incident/incident.controller.ts` (class `IncidentController`)
- `src/incident/incident.service.ts` (class `IncidentService`, method `getSummary()`)

**ไฟล์ที่ต้องแก้ไข:**

- `src/app.module.ts` (import และเพิ่ม `IncidentController`, `IncidentService`)

**เงื่อนไข:**

- ใน `IncidentService` ให้จำลอง Array ของเหตุการณ์ (Mock Data) ดังนี้:
  ```typescript
  private readonly incidents = [
    { id: 1, incidentName: 'KEYLOGGER', status: 'ACTIVE' },
    { id: 2, incidentName: 'ZERO_DAY', status: 'RESOLVED' },
    { id: 3, incidentName: 'RANSOMWARE', status: 'IGNORED' },
    { id: 4, incidentName: 'DATA_BREACH', status: 'ACTIVE' },
    { id: 5, incidentName: 'MALWARE', status: 'RESOLVED' },
    { id: 6, incidentName: 'PHISHING', status: 'RESOLVED' }
  ];
  ```
- Method `getSummary()` จะต้องไม่คืนค่า Array ออกไปตรงๆ แต่ให้ทำการรวบรวม (Aggregate) จำนวนสถานะแต่ละแบบจาก mock data ข้างต้น
- ส่งผลลัพธ์กลับไปเป็น object ที่มี key เป็นชื่อสถานะ และ value เป็นจำนวนนับ

**ตัวอย่างการเรียก API:**

```bash
curl http://localhost:3000/incidents/summary
```

**ตัวอย่างผลลัพธ์:**

```json
{
  "ACTIVE": 2,
  "RESOLVED": 3,
  "IGNORED": 1
}
```

**หมายเหตุ:**

- ข้อนี้ต้องการทดสอบ Logic การวนลูปหรือใช้ Array helper (เช่น `.reduce()`) เพื่อปรับแต่งข้อมูลก่อนส่งออก

**ข้อห้าม:**

1. ห้ามใช้ `process.exit()` มิเช่นนั้นจะตัดคะแนนเป็น 0 ทันที
2. ห้ามแก้ไข file `incidents.json` หากใช้เพื่อทดสอบเสร็จแล้วให้แก้ไขเป็นข้อมูลเดิมหรืออย่า push ขึ้น git repository.
