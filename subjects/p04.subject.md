# โจทย์ที่ 4: การอนุมัติและสร้างเหตุการณ์ใหม่ (Basic POST Request)

**Path:** `POST /incidents`

**Body:** JSON object ของเหตุการณ์ใหม่

**ผลลัพธ์:** สร้างเหตุการณ์ใหม่พร้อม `id` ที่เป็น auto-increment และบันทึกลงไฟล์ `data/incidents.json` พร้อม return ข้อมูลเหตุการณ์ที่เพิ่งสร้าง

**ไฟล์ที่ต้องแก้ไข (ไฟล์ที่มีอยู่แล้วจากข้อ 1-3):**

- `src/incident/incident.service.ts` (เพิ่ม method `create(body: any)` เพื่อสร้างเหตุการณ์ใหม่และบันทึกลงไฟล์)
- `src/incident/incident.controller.ts` (เพิ่ม endpoint `POST /incidents` รับค่าด้วย `@Body()`)

**ตัวอย่างการเรียก API:**

```bash
curl -X POST http://localhost:3000/incidents \
  -H "Content-Type: application/json" \
  -d '{
    "incidentName": "CROSS_SITE_SCRIPTING",
    "severity": "HIGH",
    "attackerIp": "Evil Hacker",
    "detectedAt": "2024-05-01"
  }'
```

```json
{
  "id": "3",
  "incidentName": "CROSS_SITE_SCRIPTING",
  "status": "ACTIVE",
  "severity": "HIGH",
  "attackerIp": "Evil Hacker",
  "detectedAt": "2024-05-01",
  "resolvedAt": null
}
```

**หมายเหตุเพิ่มเติม:**

- ข้อนี้เน้นการรับข้อมูลจาก Request Body และการทำงานของ Method `POST` ขั้นพื้นฐานแบบไม่ใช้ `ValidationPipe`
- ข้อมูลใหม่ที่ต้องบันทึกลงไฟล์จะต้องมีโครงสร้างเหมือนกับที่มีอยู่ และเซ็ตค่า default ให้ฟิลด์ที่ขาดหายดังนี้:
  - กำหนด `status` เป็น `"ACTIVE"` โดยอัตโนมัติ
  - กำหนด `resolvedAt` เป็น `null` โดยอัตโนมัติ
  - สร้าง `id` เป็นเลขหมายถัดจากรายการสุดท้ายใน data (เช่น ถ้า id ล่าสุดคือ "2", อันใหม่ต้องเป็น "3")
- ต้องเขียนข้อมูลที่มีการเพิ่มรายการใหม่ลงทับในไฟล์ `data/incidents.json`

**ข้อห้าม:**

1. ห้ามใช้ `process.exit()` มิเช่นนั้นจะตัดคะแนนเป็น 0 ทันที
2. ห้ามแก้ไข file `incidents.json` หากใช้เพื่อทดสอบเสร็จแล้วให้แก้ไขเป็นข้อมูลเดิมหรืออย่า push ขึ้น git repository.
