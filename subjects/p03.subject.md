# โจทย์ที่ 3: การปกปิดข้อมูลขั้นสูงตามระดับสิทธิ์ (Role-based Data Redaction)

**Path:** `GET /incidents/:id`

**Query Parameters:** `clearance` (string) - ระบุระดับสิทธิ์ (เช่น `STANDARD`, `SECRET`, `TOP_SECRET`) (ไม่จำเป็นต้องระบุก็ได้ ค่าเริ่มต้นคือ `STANDARD`)

**ผลลัพธ์:** ข้อมูลเหตุการณ์ 1 รายการ พร้อมการเซ็นเซอร์บางฟิลด์ หากระดับสิทธิ์ไม่ถึง

**ไฟล์ที่ต้องแก้ไขเพิ่มเติม:**

- `src/incident/incident.service.ts` (เพิ่ม method `findOne(id: string, clearance: string)`)
- `src/incident/incident.controller.ts` (เพิ่ม endpoint `GET /incidents/:id` พร้อมรับ query `clearance`)

**ข้อมูลอ้างอิง:**

- **ระดับความเสี่ยง (`severity`) ที่เป็นไปได้ทั้งหมด:** `"LOW"`, `"MEDIUM"`, `"HIGH"`, `"CRITICAL"`
- **ระดับสิทธิ์ (`clearance`) ที่เป็นไปได้ทั้งหมด:** `"STANDARD"`, `"SECRET"`, `"TOP_SECRET"`

**เงื่อนไข:**

- อ่านข้อมูลจาก `data/incidents.json` เพื่อหาเหตุการณ์ที่มี `id` ตรงกัน
- หากหาไม่พบ ให้ throw `NotFoundException` (404)
- **เงื่อนไขเซ็นเซอร์ (Data Redaction):**
  - หากเหตุการณ์นั้นมี `severity` เป็น `"HIGH"` หรือ `"CRITICAL"` แต่ query `clearance` เป็นอย่างอื่นที่ไม่ใช่ `"TOP_SECRET"` ให้เปลี่ยนค่าฟิลด์ `attackerIp` ใน response เป็น `"***REDACTED***"`

**ตัวอย่างการเรียก API:**

1. ดึงข้อมูลเหตุการณ์ทั่วไป (MEDIUM) สิทธิ์ไหนก็ดู target ได้

```bash
curl http://localhost:3000/incidents/1?clearance=STANDARD
```

```json
{
  "id": "1",
  "incidentName": "SQL_INJECTION",
  "status": "RESOLVED",
  "attackerIp": "192.168.1.10",
  "severity": "MEDIUM"
  ...
}
```

2. ดึงข้อมูลเหตุการณ์อันตราย (HIGH) แต่สิทธิ์แค่ SECRET

```bash
curl "http://localhost:3000/incidents/2?clearance=SECRET"
```

```json
{
  "id": "2",
  "incidentName": "DDOS_ATTACK",
  "status": "ACTIVE",
  "attackerIp": "***REDACTED***",
  "severity": "HIGH"
  ...
}
```

3. ดึงข้อมูลเหตุการณ์อันตราย (HIGH) ด้วยสิทธิ์ TOP_SECRET

```bash
curl "http://localhost:3000/incidents/2?clearance=TOP_SECRET"
```

```json
{
  "id": "2",
  "incidentName": "DDOS_ATTACK",
  "status": "ACTIVE",
  "attackerIp": "Unknown Botnet",
  "severity": "HIGH"
  ...
}
```

**ข้อห้าม:**

1. ห้ามใช้ `process.exit()` มิเช่นนั้นจะตัดคะแนนเป็น 0 ทันที
2. ห้ามแก้ไข file `incidents.json` หากใช้เพื่อทดสอบเสร็จแล้วให้แก้ไขเป็นข้อมูลเดิมหรืออย่า push ขึ้น git repository.
