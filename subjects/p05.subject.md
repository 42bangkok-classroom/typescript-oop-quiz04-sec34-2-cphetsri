# โจทย์ที่ 5: การยกเลิกเหตุการณ์ (DELETE Request)

**Path:** `DELETE /incidents/:id`

**ผลลัพธ์:** ลบเหตุการณ์จากไฟล์ `data/incidents.json` พร้อมส่ง API Response กลับมาแจ้งสถานะ

**ไฟล์ที่ต้องสร้าง/แก้ไข (ไฟล์ที่มีอยู่แล้วจากข้อ 1-4):**

- `src/incident/incident.service.ts` (เพิ่ม method `remove(id: string)` เพื่อลบข้อมูลเหตุการณ์)
- `src/incident/incident.controller.ts` (เพิ่ม endpoint `DELETE /incidents/:id`)

**เงื่อนไข:**

- รับ `id` ของเหตุการณ์ที่ต้องการลบผ่านทาง URL Parameter (`@Param()`)
- อ่านข้อมูลทั้งหมดจาก `data/incidents.json` มาค้นหาว่ามีเหตุการณ์ที่มี `id` ตรงกันหรือไม่
- หากหา **ไม่พบ** ให้ throw `NotFoundException` (404 status code)
- หากหา **พบ** ให้ทำการลบเหตุการณ์นั้นออกจาก Array
- เขียนข้อมูลตัวใหม่ที่ถูกลบรายการนั้นออกไปแล้ว ลงทับกลับไปในไฟล์ `data/incidents.json`
- รีเทิร์น Object แจ้งเตือนสถานะสำเร็จกลับไป (200 OK)

**ตัวอย่างการเรียก API:**

1. ลบเหตุการณ์ที่หาพบสำเร็จ:

```bash
curl -X DELETE http://localhost:3000/incidents/1
```

```json
{
  "message": "Incident ID 1 has been successfully deleted."
}
```

2. ลบเหตุการณ์ที่ไม่มีอยู่จริง:

```bash
curl -X DELETE http://localhost:3000/incidents/999
```

```json
{
  "statusCode": 404,
  "message": "Not Found",
  "error": "Not Found"
}
```

**หมายเหตุเพิ่มเติม:**

- ข้อนี้เน้นการจัดการ HTTP Method `DELETE`
- ต้องการให้ผู้เรียนรู้จักการเขียนแก้ไขไฟล์ภายนอก (Write File) ควบคู่กับการประมวลผล Array รูปแบบพื้นฐาน (เช่น `filter()` หรือ `splice()`)

**ข้อห้าม:**

1. ห้ามใช้ `process.exit()` มิเช่นนั้นจะตัดคะแนนเป็น 0 ทันที
2. ห้ามแก้ไข file `incidents.json` หากใช้เพื่อทดสอบเสร็จแล้วให้แก้ไขเป็นข้อมูลเดิมหรืออย่า push ขึ้น git repository.
